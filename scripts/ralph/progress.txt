## Codebase Patterns
- Ads were used via `@/components/ads` barrel export and `env.NEXT_PUBLIC_*_AD_SLOT` env vars
- Navigation tabs defined in `src/features/layout/BottomNavigation.tsx` and `src/features/layout/Header.tsx`
- Premium system was in `src/shared/lib/premium/`, `src/features/premium/`, `src/components/premium/`
- The project uses a linter that auto-removes unused imports on save
- Use `pnpm exec tsc --noEmit` for typecheck (need `pnpm install` first if no node_modules)
- Pre-existing typecheck errors exist in `.next/types/validator.ts` (layout type mismatches) and `@public` image imports — these are not regressions
- Env vars defined in `src/env.ts` using `@t3-oss/env-nextjs` with zod schemas
- Sitemap is in `app/sitemap.ts`
- Use `NEXT_TURBOPACK=0 npx next build` for reliable builds on Windows
- The `import/order` eslint rule: no empty lines within groups, one empty line between groups
