## Codebase Patterns
- Feature-Sliced Design (FSD) is used for code organization: features/, entities/, shared/
- Bottom navigation tabs are defined in src/features/layout/BottomNavigation.tsx
- API routes live under app/api/ and may import from feature directories
- Exercise statistics API routes are under app/api/exercises/[exerciseId]/statistics/
- When removing a feature directory, check for API routes that import from it and preserve unrelated routes in the same directory
- Stale .next/types can cause phantom TS errors after deleting routes — delete .next/types to clear them
- ExerciseAttributeValueEnum was used by Program.type, Program.equipment, ProgramSession.equipment, WorkoutSession.muscles — all changed to String/String[]
- attribute-value-translation.ts and equipment-config.ts define local string-based translations/configs that replaced Prisma enum imports
